# 贡献者激励细则

## 一、总则

本细则是对[《贡献者激励》](./contributor_role.md)的具体化与补充（试运行），旨在明确各类荣誉的认定流程、标准、权益兑现方式及退出机制，确保激励体系的公平、透明和可执行。

## 二、贡献者 (Contributor)

贡献者是``Kuikly``社区最基础的荣誉身份，旨在认可所有形式的有效贡献。

### 2.1 认定标准

任何遵守[Kuikly社区行为准则](../../CODE_OF_CONDUCT.md)，并完成以下任一有效贡献的个体，即可被视为``Kuikly``贡献者：

* **代码/PR类**：提交的Pull Request被项目主仓库或官方子仓库合并。
* **组件类**：提交的共享组件被``Kuikly``官方生态仓库收录。
* **知识分享类**：提交的文档修改被合并；在``Kuikly``指定或认可的平台发表技术教程、实践经验等并获得一定互动。
* **问题与建议**：在GitHub提交的Bug报告或产品建议被标记为``accepted``或``planned``。
* **社区活动**：成功组织或作为主要分享者参与``Kuikly``社区认可的线上/线下活动。

### 2.2 荣誉与权益

* **社区名录**：您的名字（或GitHub ID）将被列入项目主仓库的 ``README.md`` 以及官方网站的贡献者名录中，获得永久认可。

## 三、贡献者先锋 (Pioneer Contributor)

贡献者先锋是授予持续杰出贡献者的高级季度荣誉。每季度根据贡献者的综合表现进行排名，并在设定的荣誉席位上限内授予。本荣誉动态评定，旨在激励社区成员持续深度参与。

### 3.1 通用评定流程（季度）

1. **数据采集**：系统/核心团队收集所有社区成员本季度的贡献数据。
2. **进入候选池**：贡献数据达到**基础贡献线**的成员，进入本类别先锋的候选池。
3. **综合评分与排名**：核心团队依据**贡献度评价维度**对候选池成员进行综合评分，并生成排名。
4. **荣誉授予**：根据排名先后，授予**席位上限**内的成员"贡献者先锋"头衔及权益。如候选人数不足上限，则按实际达标人数授予。

### 3.2 各类先锋具体细则

#### 代码PR先锋

| 项目 | 内容 |
| --- | --- |
| **基础贡献线（候选池门槛）** | 季度内合并PR ≥ 1个 |
| **贡献度评价维度（用于排名，权重由核心团队核定）** | 1. PR合并数量与代码行数（合理加权）<br>2. 解决Issue的优先级与数量（High/Critical权重高）<br>3. Code Review参与度（评论数量与质量）<br>4. PR质量（通过率、测试覆盖、架构设计） |
| **席位上限（每季度）** | 3-5名 |
| **质量与影响力要求** | 代码符合规范，设计合理，积极协作。无重大质量问题。 |
| **核心荣誉与权益** | 1. **专属荣誉**："代码PR先锋"徽章，官网荣誉墙展示。<br>2. **深度参与**：经评估可获指定模块PR合并权限。<br>3. **优先响应**：特性需求进入优先评估通道。<br>4. **核心交流**：加入先锋群，参与路线图讨论。 |

#### 组件共创先锋

| 项目 | 内容 |
| --- | --- |
| **基础贡献线（候选池门槛）** | 季度内有组件被收录或已有组件获新Star |
| **贡献度评价维度（用于排名，权重由核心团队核定）** | 1. 组件被收录/更新次数<br>2. 组件受欢迎度（Star数、下载量、被引用数）<br>3. 组件维护活跃度（Issue响应速度、版本更新）<br>4. 组件质量（文档完整性、设计规范性、测试覆盖） |
| **席位上限（每季度）** | 3-5名 |
| **质量与影响力要求** | 组件设计规范，文档齐全，维护响应及时。 |
| **核心荣誉与权益** | 1. **专属荣誉**："组件共创先锋"徽章，官网荣誉墙展示。<br>2. **生态推广**：组件可入选"官方推荐库"，获重点推广。<br>3. **优先支持**：享有技术支持快速通道。<br>4. **核心交流**：加入先锋群，参与生态决策。 |

#### 社区分享先锋

| 项目 | 内容 |
| --- | --- |
| **基础贡献线（候选池门槛）** | 季度内发布1篇被认可文章或完成1次社区分享 |
| **贡献度评价维度（用于排名，权重由核心团队核定）** | 1. 内容产出量（文章、教程、视频数量）<br>2. 内容影响力（阅读量、点赞、收藏、评论互动）<br>3. 知识支持度（官方答疑被采纳数、帮助他人次数）<br>4. 活动组织与参与（主讲次数、活动规模与反馈） |
| **席位上限（每季度）** | 3-5名 |
| **质量与影响力要求** | 内容准确、有深度，对社区有积极带动作用。 |
| **核心荣誉与权益** | 1. **专属荣誉**："社区分享先锋"徽章，官网荣誉墙展示。<br>2. **舞台赋能**：优先获荐为官方或合作大会讲师。<br>3. **内容共创**：优质内容被官方转载，可合作策划专题。<br>4. **核心交流**：加入先锋群，参与社区运营讨论。 |

#### 先锋通用权益

所有先锋均享有每季度获得**限量定制礼品**及参与**"年度杰出先锋"评选**的权益。

## 四、评定与执行流程

### 4.1 流程概览

```
提名/申请 → 收集材料 → 核心团队评估 → 结果公示 → 权益授予
```

### 4.2 具体步骤

**提名与申请（每季度最后一个月1-10日）**：

* **核心团队提名**：核心团队根据后台数据（GitHub Insights、社区数据等）初步筛选候选人，并主动发出邀请。
* **自助申请**：有意愿的开发者也可通过 Readme ``Online Support`` 主动申请，经核心团队观察评估后纳入候选池。

**材料审核与评估（每季度最后一个月11-20日）**：

由核心团队组成评定小组，根据本细则标准，结合量化数据和贡献质量进行综合评估。

**结果公示（每季度最后一个月25日前）**：

在GitHub Discussions的"Announcements"板块发布新一期"贡献者先锋"名单，公示7天。无重大异议则生效。

**权益授予（新季度首月第一周完成）**：

* 更新官网和 ``README.md`` 名单。
* 邀请新晋先锋加入专属交流群。
* 联系先锋寄送礼品、沟通权限授予事宜。

## 五、退出机制

为保证先锋社区的活力与先进性，设立如下机制：

1. **不活跃状态**：若贡献者先锋连续**两个季度**未能达到其对应类别的**基础贡献线**，且未与核心团队说明情况，则被视为"不活跃"。
2. **预警沟通**：在首个季度未达标时，核心团队将进行友好沟通与提醒。
3. **荣誉转移**：若连续两个季度不活跃，其先锋头衔及附属权益将被暂停，并从当前名单中移至"历史荣誉贡献者"名录，以感谢其历史贡献。
4. **复活跃途径**："历史荣誉贡献者"重新开始贡献并达到一个季度的评定标准后，可快速恢复先锋身份。
